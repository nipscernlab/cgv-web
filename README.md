# CGV-WEB — ATLAS Calorimeter Geometry Viewer (Web Edition)

A high-performance, client-side 3D visualisation tool for ATLAS calorimeter collision data.  
Built with **Rust/WASM** (geometry engine) + **Three.js** (WebGL rendering) + **Vite** (build).

---

## Architecture

```
XML drag-drop
      │
      ▼
 ┌─────────────────────────────────────────────────────┐
 │  Rust / WASM  (src/lib.rs)                          │
 │  ① quick-xml streaming parser → HashMap pile-up     │
 │  ② Geometry lookup (CaloGeoConst.h constants)       │
 │  ③ Cell centre + box dimensions (CaloBuild.C logic) │
 │  ④ Column-major 4×4 instance matrices + RGB colours │
 │     → Float32Arrays returned to JS                  │
 └─────────────────────────────────────────────────────┘
      │
      ▼
 ┌─────────────────────────────────────────────────────┐
 │  Three.js  (main.js)                                │
 │  ⑤ InstancedMesh (single draw-call, up to 200k cells│
 │  ⑥ OrbitControls: orbit · zoom · pan               │
 │  ⑦ North/South axis markers + energy legend        │
 │  ⑧ PNG screenshot download                        │
 └─────────────────────────────────────────────────────┘
```

## Layer index mapping (XML `l=` attribute)

| XML `l` | Sub-detector | Layers | η cells | φ cells |
|---------|--------------|--------|---------|---------|
| 0       | Tile A barrel | h=2300–2600 mm | 10 | 64 |
| 1       | Tile B+C barrel (merged) | h=2600–3440 mm | 9 | 64 |
| 2       | Tile D barrel | h=3440–3820 mm | 4 | 64 |
| 3–5     | Tile A/B/D extended | — | 5,5,2 | 64 |
| 6–13    | Tile scint/MBTS | — | 1 each | 64/8 |
| 14      | HEC layer 1 | z=4350–4630 mm | 14 | 64 |
| 15–17   | HEC layers 2–4 (merged pairs) | — | 13,12,12 | 64 |
| 18–21   | LAr EM Barrel layers 0–3 | r=1422–1985 mm | 61/451/57/27 | 64/64/256/256 |
| 22–25   | LAr EM EndCap layers 0–3 | z=3681–4243 mm | 12/216/51/34 | 64/64/256/256 |

**Eta index convention:** positive `eta` → positive Z (North/Side A); negative `eta` → negative Z (South/Side C).  
Energy pile-up: multiple `<cell>` entries with the same `(l, eta, phi)` are summed automatically.

## Quick Start

```bash
# 1. Install JS dependencies
npm install

# 2. Build the Rust → WASM package
wasm-pack build --target web --out-dir pkg

# 3. Start dev server
npm run dev
# → open http://localhost:5173
```

For a production build:
```bash
npm run build      # outputs to dist/
npm run preview    # preview the built site
```

## Usage

1. Open the app in your browser.
2. **Drag & drop** (or click) to load an ATLAS event XML file.  
   Format: `<cell l="0" eta="3" phi="12" e="42.5" />`
3. Interact with the 3D view:
   - **Left drag** — orbit
   - **Right drag / two-finger** — pan
   - **Scroll** — zoom
4. Click **⬇ SNAPSHOT** to download a PNG of the current view.
5. Click **↺ RESET** to clear and load a new event.

## Geometry Fidelity Notes

- **Tile cells** are rendered as axis-aligned rectangular boxes with correct radial depth `(h2−h1)`, z-extent from `Tilez`/`Tiledz` tables, and phi arc-width `= R·Δφ`.  
- **HEC cells** use the `HECz` / `HECdz` lookup tables for radial position in the endcap; z-centre = `(h1+h2)/2`.  
- **LAr EM Barrel** computes the cell z-centre analytically: `z = R·tan(η₂rad(η))` matching CaloBuild.C.  
- **LAr EM EndCap** uses `LaEb_h1`/`LaEb_h2` for z-extent and `r⊥ = h1/tan(η₂rad(η))` for transverse position, exactly as in `BuildEMEndCap`.  
- Merged layers (MergeTile, MergeHEC) use the primary sub-row position for each eta cell.  
- All units internally are **mm**; Three.js units are **metres** (scale factor 0.001).
